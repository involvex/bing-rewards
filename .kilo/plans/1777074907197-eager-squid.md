# Headless Mode Implementation Plan

## Current Limitations
The bing-rewards tool currently cannot run in headless mode because:
1. It uses `pynput` to simulate keyboard input, which requires a GUI environment
2. It opens a visible Chrome window for user interaction
3. The README explicitly states it's not intended for headless operation

## Proposed Solution
To enable headless mode, we need to:
1. Add a `--headless` command-line argument
2. Modify browser launch to include headless flags when enabled
3. Replace pynput keyboard automation with direct HTTP requests or Selenium WebDriver
4. Maintain backward compatibility with existing GUI mode

## Implementation Steps

### 1. Add Headless Option
In `options.py`:
- Add `--headless` flag to argument parser
- Update Config class to include headless setting
- Modify `get_options()` to handle the new flag

### 2. Modify Browser Launch
In `app.py`:
- Update `browser_cmd()` function to add `--headless` or `--headless=new` flag when headless mode is enabled
- Consider using `--disable-gpu` and `--no-sandbox` for better compatibility

### 3. Replace Keyboard Automation
For headless mode, replace the pynput-based search implementation:
- Option A: Use Selenium WebDriver to interact with Bing search page
- Option B: Send direct HTTP requests to Bing search API (may require handling cookies/user-agent)
- Option C: Keep browser open but use JavaScript execution via DevTools Protocol

### 4. Maintain Functionality
Ensure headless mode:
- Still performs the same number of searches
- Uses the same keyword rotation logic
- Respects search delays
- Handles ESC key interruption (may need alternative mechanism)
- Opens rewards page at end if requested

### 5. Testing
- Verify headless mode works without GUI display
- Confirm searches are properly credited to Bing Rewards account
- Ensure no regression in existing GUI mode functionality

## Files to Modify
- `bing_rewards/options.py` - Add headless argument
- `bing_rewards/app.py` - Modify browser launch and search implementation
- `README.md` - Update documentation to mention headless capability

## Dependencies
May need to add:
- selenium (if using WebDriver approach)
- or ensure existing dependencies support headless operation

## Estimated Effort
- 2-4 hours for implementation
- 1 hour for testing
- 30 minutes for documentation updates
