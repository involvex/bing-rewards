## Repository Analysis
- Python CLI tool for automating Bing searches to earn Microsoft Rewards points
- Uses uv for dependency management and packaging
- Main entry point: `bing_rewards.app:main`
- Key dependencies: pynput for keyboard control
- Requires Chrome/Chromium browser and logged-in Bing/Microsoft account

## Essential Information for AGENTS.md

### Development Setup
1. Install uv (package manager)
2. Clone repository
3. Run `uv run pre-commit install --install-hooks` to set up pre-commit hooks
4. Run `uv run bing-rewards --help` to verify installation and see options

### Common Commands
- Development run: `uv run bing-rewards [options]`
- Run linter: `uv run ruff check`
- Run formatter: `uv run ruff format --check`
- Run tests: [No explicit test suite found - relies on manual verification]

### Important Conventions
- Config file location: `%APPDATA%\bing-rewards\config.json` (Windows) or `$XDG_CONFIG_HOME/bing-rewards/config.json`
- Default user agents mimic Microsoft Edge to avoid detection
- ESC key exits early during execution
- No other Chrome instances should be running when script executes
- Uses threading for search execution with keyboard listener for ESC

### CI/CD Processes
- Linting: GitHub Actions runs ruff on PRs to master
- Release: Automated via release-please-action
  - Push to master triggers release process
  - Test PyPI publish on PR creation
  - PyPI publish on release creation

### Architecture Notes
- Main logic in `bing_rewards/app.py`
- Configuration and argument parsing in `bing_rewards/options.py`
- Keywords sourced from `bing_rewards/data/keywords.txt`
- Uses pynput for actual keyboard input to perform searches
- Supports multiple Chrome profiles run sequentially
- Separate desktop and mobile search modes with different user agents

### Gotchas
- Must have Chrome/chromium executable in PATH or specify with `--exe`
- Must be logged into Bing.com at least once to save cookies
- Warning about effectiveness with newer Bing systems (see README)
- Search delay configuration is important to avoid detection
- Dry run mode (`--dryrun`) useful for testing without actual searches
